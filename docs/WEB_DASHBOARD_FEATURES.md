# Web Dashboard 前端功能说明

## 新增功能：批量导入 & 自动同步

### 1. 批量导入 (📥 按钮)

#### 位置
顶部导航栏，"立即刷新" 按钮旁边

#### 功能
从 Elasticsearch 批量导入历史 traces 到本地

#### 使用流程
1. 点击 "📥 批量导入" 按钮
2. 弹窗中设置参数：
   - **开始时间**：默认 7 天前
   - **结束时间**：默认当前时间
   - **导入数量**：最多导入多少条（默认 1000）
   - **输出文件**：保存位置（默认 data/agent_runs.json）
3. 点击 "开始导入"
4. 实时显示导入进度条
5. 导入完成后自动刷新页面

#### 示例
```
场景：导入最近 7 天的数据
1. 点击 "📥 批量导入"
2. 保持默认日期范围（自动设置为 7 天前 ~ 现在）
3. 设置导入数量为 1000
4. 点击 "开始导入"
5. 进度条显示进度（0% → 100%）
6. 导入完成，自动刷新 Runs 列表
```

### 2. 自动同步 (🔄 按钮)

#### 位置
顶部导航栏，"批量导入" 按钮旁边

#### 功能
启动后台线程定时从 Elasticsearch 拉取最新 traces

#### 界面
点击 "🔄 自动同步" 后，右下角弹出浮动面板，显示：
- **状态**：🟢 运行中 / 🔴 未运行
- **最后同步**：上一次同步的时间戳
- **累计导入**：总共导入了多少条 traces
- **错误数**：同步过程中遇到的错误数

#### 配置项
| 参数 | 默认值 | 说明 |
|------|--------|------|
| 同步间隔（秒） | 300 | 每 300 秒（5 分钟）拉取一次 |
| 回溯窗口（分钟） | 60 | 每次只拉取最近 60 分钟的数据 |
| 输出文件路径 | data/agent_runs.json | 同步结果保存位置 |

#### 使用流程
1. 点击 "🔄 自动同步"，右下角出现控制面板
2. 调整参数（可选）
3. 点击 "启动同步"
4. 面板显示：
   - 状态变为 🟢 运行中
   - "启动同步" 按钮禁用
   - "停止同步" 按钮启用
5. 后台每 5 分钟自动拉取一次数据
6. 点击 "刷新状态" 实时查看统计
7. 点击 "停止同步" 停止后台线程

#### 示例工作流
```
时间线：
09:00 - 点击 "🔄 自动同步"，启动同步
09:05 - 第 1 次同步，导入 50 条 traces
09:10 - 第 2 次同步，导入 30 条 traces（去重）
09:15 - 点击 "刷新状态"，显示：累计 80 条，错误 0
09:20 - 继续后台同步...
10:00 - 点击 "停止同步"，结束
```

## 前端代码实现

### HTML 新增元素

#### 按钮（顶部导航栏）
```html
<button id="importBtn" class="primary">📥 批量导入</button>
<button id="syncBtn" class="primary">🔄 自动同步</button>
```

#### 批量导入弹窗 (id="importModal")
- 表单输入：日期、limit、输出路径
- 进度显示：进度条 + 状态文本
- 操作按钮：开始、取消

#### 自动同步面板 (id="syncPanel")
- 状态显示：4 个统计数字
- 配置输入：3 个参数
- 操作按钮：启动、停止、刷新

### CSS 样式

#### 新增 class
- `.primary`：深蓝色按钮（导入和同步）
- `.modal` / `.modal-content`：弹窗容器
- `.sync-panel`：右下角浮动面板
- `.form-group`：表单组件
- `.progress-bar`：进度条

#### 响应式设计
- PC：导入弹窗（500px）、同步面板（320px）
- 移动端：待完善（可选）

### JavaScript 实现

#### 导入流程
```javascript
1. initImportModal()        - 初始化弹窗事件
2. startImport()            - 启动导入，调用 /api/import/start
3. 轮询 /api/import/<id>/status（每 1 秒）
4. 显示进度条和状态文本
5. 完成后自动刷新页面
```

#### 同步流程
```javascript
1. initSyncPanel()          - 初始化面板事件
2. startSync()              - 启动同步，调用 /api/sync/start
3. stopSync()               - 停止同步，调用 /api/sync/stop
4. updateSyncStatus()       - 查询状态，调用 /api/sync/status
5. 更新面板中的状态显示
```

## 常见问题

### Q: 批量导入和自动同步的区别？
**A:**
- **批量导入**：一次性导入历史数据，适合初始化
- **自动同步**：后台定时拉取新数据，适合持续更新

### Q: 导入过程中可以关闭弹窗吗？
**A:** 不建议。导入过程中关闭弹窗会丢失进度信息，但后台导入仍在继续。可以稍后通过查询 task_id 重新获取状态。

### Q: 自动同步会占用很多资源吗？
**A:** 不会。同步间隔默认 5 分钟，每次查询很小（只查询最近 1 小时的数据），不影响主页面性能。

### Q: 如何禁用自动刷新？
**A:** 在 "自动刷新" 复选框中取消勾选。

### Q: Traces 导入后在哪里看？
**A:** 导入完成后，右侧 "Runs" 列表会自动刷新，显示新导入的 traces。

## 技术细节

### 默认日期计算
```javascript
// 导入弹窗打开时，自动计算：
- 开始时间 = 当前时间 - 7 天
- 结束时间 = 当前时间
```

### 进度轮询机制
```javascript
// 导入时轮询状态
interval: 1 秒
max_attempts: 600（10 分钟超时）
status: running|done|error
```

### 按钮禁用状态
```javascript
同步运行中：
  启动同步 按钮：disabled = true
  停止同步 按钮：disabled = false

同步未运行：
  启动同步 按钮：disabled = false
  停止同步 按钮：disabled = true
```
